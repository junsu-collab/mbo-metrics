# MBO Metrics

뉴스디자인팀 업무평가를 위한 웹 앱입니다. 팀원 개인의 업무 수행 결과를 난이도·기여도·수행점수로 수치화해서, 주관적 인상이 아닌 공정하고 투명한 기준으로 평가할 수 있게 도와줍니다.

## 평가 체계

MBO(Management By Objectives) 방식을 기반으로 한 정량 평가 시스템입니다.

- **객관성** — 개인의 감정이나 인상이 아닌 수치 기반으로 평가
- **투명성** — 점수 산출 공식과 계수를 사전에 공개
- **형평성** — 모든 팀원에게 동일한 기준을 적용
- **발전성** — 평가 결과는 처벌이 아닌 성장의 근거로 활용

### MBO 항목 구성

평가는 총 6개의 MBO 항목, 합계 100점으로 구성됩니다.

| 구분 | 항목명 | 배점 | 비고 |
|---|---|---|---|
| 공통 | 제작안정성 | 10점 | 모든 팀원 동일 |
| 공통 | 조직강화기여 | 10점 | 모든 팀원 동일 |
| 공통 | 뉴스그래픽 범위와 완성도 | 40점 | 모든 팀원 동일 |
| 선택 | 디자인 개발 | 자율 배분 | 합계 40점 |
| 선택 | 빅이벤트 지원 | 자율 배분 | 선택 항목 중 |
| 선택 | 선거방송 대응 | 자율 배분 | 최대 3개 활용 |

공통 항목 3개(60점)는 모든 팀원에게 동일하게 적용되고, 선택 항목 3개(40점)는 팀원 개인의 주요 역할에 따라 자율적으로 배점을 나눕니다(예: 선거방송 대응 30점 + 빅이벤트 지원 10점 = 40점).

### 점수 산출 공식

```
항목 점수 = (W합계 × S ÷ 기준 상한) × 항목 배점

W = 난이도계수 × 기여도계수
S = 팀장점수 × 70% + 팀원점수 × 30%
기준 상한 = 난이도 최고 × 기여도 최고 × 5점
```

| 변수 | 명칭 | 설명 |
|---|---|---|
| W | 업무 가중치 | 난이도계수 × 기여도계수. 업무의 복잡도와 기여 빈도를 반영 |
| S | 수행점수 | 팀장(70%)·팀원(30%) 수행점수(1~5점)의 가중 합산값 |
| 기준 상한 | 정규화 기준 | 난이도 최고 × 기여도 최고 × 5점. 점수를 0~1 비율로 정규화하는 분모 |
| 항목 배점 | MBO 배점 | 해당 MBO 항목에 할당된 배점 |

**난이도 계수** (5단계, 곱셈 특성상 범위를 좁게 유지)

| 단계 | 정의 | 계수 |
|---|---|---|
| 단순 | 매뉴얼에 따른 반복 | ×0.95 |
| 일반 | 적정 수준의 실무 | ×1.00 |
| 복합 | 다각적 접근 활용 | ×1.05 |
| 전문 | 고도의 지식과 스킬 요구 | ×1.10 |
| 전략 | 독자 판단 및 방향 설정 | ×1.15 |

**기여도 계수** (3단계, 주간업무보고를 통한 공유 빈도 기준)

| 단계 | 기준 | 계수 |
|---|---|---|
| 낮음 | 미보고 / 팀 내부 업무 | ×1.00 |
| 보통 | 주간업무보고 1~2회 | ×1.10 |
| 높음 | 주간업무보고 3회 이상 | ×1.20 |

**수행점수(S)**는 업무별이 아니라 MBO 항목별로 한 번만 입력합니다. 같은 항목에 여러 업무가 있으면, 각 업무의 영향력 비중을 10% 단위로 설정할 수 있고(미설정 시 균등 배분) 이 비중이 W합계 계산에 반영됩니다.

| 점수 | 수행 수준 |
|---|---|
| 5점 | 목표를 크게 초과 달성, 탁월한 성과 |
| 4점 | 목표 초과 달성, 우수한 성과 |
| 3점 | 목표 달성, 기대 수준 충족 |
| 2점 | 목표 일부 미달, 개선 필요 |
| 1점 | 목표 미달, 상당한 개선 필요 |

이 계산 로직은 `src/lib/calc.ts`에 순수 함수로 구현돼 있고, 원문 계산 예시(W합계 1.266 · S 3.70 · 기준상한 6.90 → 약 27점)와 대조 검증했습니다.

## 주요 기능

- **팀원·업무 관리** — 팀원을 추가하고 업무를 정의(업무명·MBO 항목·난이도·기여도). 다른 팀원이 등록한 유사 업무명이 자동완성으로 제안되고, 선택하면 계수까지 같이 채워집니다.
- **항목별 수행점수 입력** — 팀장·팀원 점수를 선택하면 즉시 항목 점수와 달성률이 계산됩니다.
- **업무 중요도 비중** — 같은 항목에 업무가 여러 개면 슬라이더로 비중을 조정할 수 있습니다.
- **선택 항목 배점 패널** — 선택 항목 3개의 배점 합계(40점)를 10점 단위로 맞추도록 안내합니다.
- **모든 업무 보기** — 전체 팀원의 업무를 한 화면에서 검색하고, 유사한 업무끼리 자동으로 묶어서 계수를 일괄 수정할 수 있습니다.
- **순위 시뮬레이터** — 난이도·기여도 계수와 팀장/팀원 가중치를 임시로 조정하며, 실제 체감 순위와 산식 순위의 상관도(ρ, 스피어만 순위상관계수)를 실시간으로 확인할 수 있습니다. 저장을 눌러야 실제 설정에 반영됩니다.
- **평가 연도 관리** — 연도별로 데이터를 분리 관리하며, 새 연도를 추가하면 직전 연도 계수를 자동으로 복제합니다.
- **데이터 내보내기** — 개인별/연도 전체 JSON 백업, 팀원별 Excel 상세 내역, 연도 전체 종합순위 Excel을 내려받을 수 있습니다.
- **오프라인 사용 · PWA** — 서버 없이 브라우저에서 동작하고, 홈 화면에 앱으로 설치할 수 있습니다.

## 데이터 저장

데이터는 브라우저 localStorage에 자동 저장됩니다(키: `mbo_metrics_v1_0`). 별도 서버나 로그인 없이, 이 브라우저에서만 데이터가 유지됩니다. 다른 기기·브라우저로 옮기거나 백업해두려면 **JSON 내보내기(전체 팀원)** 로 파일을 저장한 뒤 **JSON 불러오기**로 복원하세요.

이전 버전(`reference/mbo_metrics_1_0.html`, 단일 HTML 파일 앱)이 저장한 데이터·백업 파일도 이 앱에서 그대로 읽을 수 있도록 마이그레이션 로직을 유지하고 있습니다.

## 개발

```bash
npm install
npm run dev       # http://localhost:3000
```

## 빌드 / 정적 검증

```bash
npm run build      # ./out 에 정적 산출물 생성
npx http-server out -p 4173 -s   # 산출물을 로컬에서 직접 서빙해 확인
```

## 배포

`main` 브랜치에 푸시되면 `.github/workflows/deploy-pages.yml`이 빌드 후 GitHub Pages로 배포합니다.
프로젝트 페이지(`https://<owner>.github.io/mbo-metrics/`) 기준으로 `NEXT_PUBLIC_BASE_PATH`를
자동으로 리포지토리 이름으로 설정합니다. 저장소 Settings → Pages에서 Source를
**GitHub Actions**로 설정해야 합니다.

커스텀 도메인이나 루트 경로에 배포하는 경우 `NEXT_PUBLIC_BASE_PATH`를 빈 문자열로 두거나
워크플로 env를 수정하세요.

## 기술 스택

Next.js(App Router) + TypeScript + Tailwind CSS + Zustand로 구성되어 있고, 서버 없이 브라우저에서 동작하는 정적 사이트(`output: 'export'`)로 빌드됩니다. 색상·폰트 등 디자인 값은 [Montage](https://github.com/wanteddev/montage-web)(Wanted Lab 디자인 시스템) 팔레트를 참고했습니다.

## 구조

- `src/lib/calc.ts` — 점수 계산 순수 함수(기존 HTML 앱과 1:1 포팅, 결과 검증됨)
- `src/lib/defaults.ts` — MBO/난이도/기여도 기본 계수
- `src/lib/similarity.ts` — 업무명 자동완성·유사 업무 그룹핑(레벤슈타인 거리)
- `src/lib/simulator.ts` — 순위 시뮬레이터 계산(스피어만 순위상관계수 등)
- `src/lib/excel.ts`, `src/lib/backup.ts` — Excel/JSON 내보내기·불러오기
- `src/store/useAppStore.ts` — Zustand + persist, 기존 localStorage 원시 포맷과 호환
- `src/components/` — 입력 패널 / 집계 패널 / 설정·시뮬레이터·모든업무 모달
- `reference/mbo_metrics_1_0.html` — 원본 단일 HTML 앱(참고용, 배포 대상 아님)
